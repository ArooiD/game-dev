# Архитектура 2.5D RTS Проекта

## Обзор

Это документ описывает архитектуру браузерной RTS игры с изометрическим видом, вдохновлённой историческими стратегиями уровня "Казаки 2".

## Структура проекта

```
src/
├── core/              # Ядро системы сущностей
│   ├── Entity.ts      # Базовый класс сущности
│   ├── EntityManager.ts # Менеджер сущностей
│   └── index.ts
├── render/            # Система рендеринга

│   └── index.ts
├── game/              # Игровая логика
│   ├── Game.ts        # Основной класс игры

├── entities/          # Конкретные сущности

```

## Основные компоненты

### 1. Система сущностей (ECS-light)

**Entity.ts**
- Базовый класс для всех игровых объектов
- Поддержка тегов и компонентов
- Иерархия родительских/дочерних сущностей
- Методы для столкновений и обновления

**EntityManager.ts**
- Управление жизненным циклом сущностей
- Индексация по типу и тегам для быстрого поиска
- Массовое обновление всех сущностей

### 2. Изометрический рендеринг

**IsoUtils.ts**
- Преобразование координат сетки ↔ экрана
- Изометрическая проекция 2:1 (угол ~26.565°)
- Размер тайла: 64×32 пикселя
- Вычисление глубины для правильной сортировки

**Camera.ts**
- Позиция и зум камеры
- Плавное перемещение
- Ограничение границами карты
- Преобразование экранных ↔ мировых координат

**Renderer.ts**
- Отрисовка на Canvas API
- Сортировка по глубине (painter's algorithm)
- Поддержка различных типов элементов:
  - Тайлы карты
  - Юниты (с полоской здоровья)
  - Здания
  - Выделение
  - Пути

### 3. Система ввода

**InputHandler.ts**
- Обработка мыши (левый/правый клик, драг)
- Выделение рамкой
- Движение камеры по краям экрана
- Скролл для зума
- Обработка клавиатуры

### 4. Игровой цикл

**GameLoop.ts**
- requestAnimationFrame цикл
- Переменный deltaTime
- Fixed update для физики
- Подсчёт FPS

**Game.ts**
- Основной контроллер игры
- Инициализация всех подсистем
- Обработка состояний (MENU, PLAYING, PAUSED, GAMEOVER)
- Управление ресурсами

### 5. Сущности

**Unit.ts**
- Типы: Infantry, Archer, Cavalry, Artillery
- Статистика: health, morale, stamina, ammo
- Механики: регенерация, получение урона

**Building.ts**
- Типы: Town Center, Barracks, Stable, Artillery Foundry, Wall
- Размеры зависят от типа
- Состояние operational

## Изометрическая проекция

```
Грид координаты → Экран:
  screenX = (gridX - gridY) * (TILE_WIDTH / 2)
  screenY = (gridX + gridY) * (TILE_HEIGHT / 2)

Экран → Грид координаты:
  gridX = (screenY / halfHeight + screenX / halfWidth) / 2
  gridY = (screenY / halfHeight - screenX / halfWidth) / 2

Глубина (для сортировки):
  depth = gridX + gridY - z
```

## Технические решения

### Почему Canvas API вместо WebGL?
- Проще для начала разработки
- Достаточно для 2D/2.5D прототипа
- Легче отлаживать
- Можно перейти на WebGL позже

### Почему не полноценный ECS?
- Для прототипа достаточно light версии
- Меньше сложности
- Быстрее итерации
- Можно refactoring позже

### Почему TypeScript?
- Типобезопасность
- Лучшая IDE поддержка
- Меньше runtime ошибок
- Легче рефакторинг
