# Contributing Guidelines

> Правила вклада в проект. Подходит для людей и AI-агентов.

## 🚀 Быстрый старт

```bash
# Установка зависимостей
npm install

# Запуск dev сервера
npm run dev

# Запуск в браузере
# http://localhost:5173
```

---

## 📝 Как делать изменения

### 1. Создайте ветку

```bash
git checkout -b feature/your-feature-name
# или
git checkout -b bugfix/fix-issue-name
```

### 2. Внесите изменения

- Следуйте правилам из `.agents/CONTEXT.md`
- Не хардкодьте арт в код
- Используйте asset registry для зданий

### 3. Проверьте работу

```bash
npm run dev
# Откройте браузер и протестируйте изменения
```

### 4. Сделайте коммит

```bash
git add .
git commit -m "feat: добавлена новая механика"
# или
git commit -m "fix: исправлена ошибка с рендерингом"
```

### 5. Отправьте изменения

```bash
git push -u origin feature/your-feature-name
```

### 6. Создайте Pull Request

- Опишите изменения
- Приложите скриншоты (если есть визуальные изменения)
- Укажите, какие задачи решены

---

## 🎯 Правила разработки

### Для AI-агентов

1. **Читайте `.agents/CONTEXT.md` перед началом работы**
2. **Не меняйте `BattleScene.ts` без явного запроса**
3. **Используйте `BuildingVisualSet` для зданий**
4. **Не добавляйте хардкод-арт в код**
5. **Делайте маленькие коммиты с понятными сообщениями**

### Для людей

1. **Пишите понятные коммиты**
   - ✅ `feat: добавлена система формаций`
   - ❌ `fix stuff`

2. **Описывайте изменения в PR**
   - Что сделано
   - Почему так
   - Как протестировать

3. **Не ломайте существующий функционал**
   - Проверяйте работу после изменений
   - Тестируйте на разных сценах

---

## 📂 Структура коммитов

### Типы коммитов

| Тип | Описание | Пример |
|-----|----------|--------|
| `feat` | Новая функция | `feat: добавлена артиллерия` |
| `fix` | Исправление бага | `fix: исправлен рендеринг крыш` |
| `docs` | Документация | `docs: обновлён README` |
| `refactor` | Рефакторинг | `refactor: разбита BattleScene` |
| `chore` | Инструменты, конфиги | `chore: обновлены зависимости` |
| `perf` | Производительность | `perf: оптимизирован рендеринг` |
| `test` | Тесты | `test: добавлены юнит-тесты` |

### Формат сообщения

```
<тип>: краткое описание

Подробное описание (если нужно):
- Что сделано
- Почему так
- Какие последствия

Fixes: #123 (если есть issue)
```

**Пример:**
```
feat: добавлена система формаций

Добавлены 4 типа формаций:
- Line: максимальный залп
- Column: быстрое движение
- Square: защита от кавалерии
- Wedge: атакующий клин

Механика:
- Скорость снижается на 20% в формации
- Бонус к бою +15% в правильной формации
- Горячие клавиши: 1-4 для смены формации

Fixes: #45
```

## 🎨 Стиль кода

### TypeScript

```typescript
// ✅ ПРАВИЛЬНО
interface UnitStats {
  health: number;
  morale: number;
}

function calculateDamage(stats: UnitStats): number {
  return stats.health * 0.1;
}

// ❌ НЕПРАВИЛЬНО
let x = {health: 100, morale: 50}
function calcDamage(s) { return s.health * 0.1 }
```

### Именование

- **Файлы:** `PascalCase.ts` (для компонентов), `camelCase.ts` (для утилит)
- **Типы:** `PascalCase` (`UnitStats`, `BuildingConfig`)
- **Функции:** `camelCase` (`calculateDamage`, `createBuilding`)
- **Константы:** `UPPER_SNAKE_CASE` (`MAX_UNITS`, `DEFAULT_SPEED`)

### Комментарии

```typescript
/**
 * Вычисляет урон на основе характеристик юнита.
 * 
 * @param stats - Характеристики юнита
 * @returns Вычисленный урон
 */
function calculateDamage(stats: UnitStats): number {
  // Базовый урон = 10% от здоровья
  return stats.health * 0.1;
}
```
