## 📖 Как использовать

### Для AI-агентов

1. **Начните с `.agents/CONTEXT.md`** — основной контекст проекта
2. **При необходимости читайте `DECISIONS.md`** — почему приняты такие решения
3. **Следуйте `.agents/CONTRIBUTING.md`** — как делать изменения
4. **Ищите TSDoc комментарии в коде** — локальный контекст функций

### Для людей

1. **Новичкам:** `.agents/CONTEXT.md` → `.agents/CONTRIBUTING.md`
2. **Для понимания архитектуры:** `DECISIONS.md`
3. **Для деталей реализации:** TSDoc в коде + `AGENTS.md`

## ⚠️ Критичные правила (кратко)

### Система зданий
- ❌ НЕ хардкодить арт в код
- ✅ Использовать `buildingAssetRegistry`
- ✅ Использовать `BuildingVisualSet` с `container root`
- ✅ Синхронизировать через `syncBuildingVisuals()`

### BattleScene.ts
- ⚠️ Файл большой и сжатый
- ❌ НЕ рефакторить без явного запроса
- ❌ НЕ удалять функционал (формации, combat, minimap)

### Рабочие
- ❌ НЕ добавлять молоток над рабочими
- ✅ Правый клик для присоединения к стройке

## 📝 TSDoc в коде

Критичные функции имеют TSDoc комментарии:
- `createBuildingVisuals()` — создание визуалов здания

**Всегда читайте TSDoc перед использованием функций!**
